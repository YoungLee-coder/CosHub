# Sections

本文件定义规则分类、排序和影响级别。
括号中的 ID 是规则文件名前缀。

---

## 1. Architecture (architecture)

**Impact:** CRITICAL
**Description:** 项目分层、数据流、SPA 路由与组件职责划分等架构约定。

## 2. Security (security)

**Impact:** CRITICAL
**Description:** 认证、授权、输入校验、敏感信息保护等安全相关的规则。

## 3. Code Quality (quality)

**Impact:** HIGH
**Description:** 代码风格、命名、错误处理、TypeScript 严格类型等可维护性标准。

## 4. Data Layer (data)

**Impact:** HIGH
**Description:** COS SDK 交互、数据模型和 fetch 封装规则。

## 5. API Design (api)

**Impact:** HIGH
**Description:** API 路由设计、请求校验、响应格式规则。

## 6. UI / Frontend (ui)

**Impact:** MEDIUM
**Description:** shadcn/ui 组件约定、Tailwind 样式规范、无障碍标准。

## 7. Performance (performance)

**Impact:** MEDIUM
**Description:** 性能优化、虚拟滚动、缓存策略。

## 8. Patterns (patterns)

**Impact:** LOW
**Description:** React Query 使用模式、TanStack Table 配置等通用模式。

## 9. Reference (reference)

**Impact:** LOW
**Description:** 关键文件位置、环境变量、本地开发指南等信息查询。
